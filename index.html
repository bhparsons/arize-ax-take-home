Deliverable ① · the build

An agent that helps devs build agents

A Developer Research Assistant: it plans a lookup, retrieves over a local doc corpus, optionally web-searches, reasons over the sources, and writes a cited answer. Built on Pydantic AI (model-agnostic) running on Claude via OpenRouter, and instrumented to Arize AX through OpenInference / OTel.

AGENT (researcher)
 ├─ LLM   plan the lookup
 ├─ TOOL  doc_lookup  →  RETRIEVER (local RAG corpus)   ← the deliberate failure surface
 ├─ TOOL  web_search
 ├─ LLM   reason over sources
 └─ LLM   writer  →  cited answer

Why Pydantic AI on Claude: it's model-agnostic, which (a) showcases Arize's OTel/OpenInference neutrality, (b) makes the required model-swap experiment sing (Claude ↔ GPT ↔ Gemini on the same agent), and (c) stays on-brand with Claude.

Deliverable ② · workflows + onboarding feedback

Both Arize workflows, evidenced

The assignment asks for the development workflow (dataset → experiment → eval) and the observability workflow (tracing + evals + a debugged issue). Both were run:

A
Development: a 12-row golden dataset → an experiment with two LLM-judges (correctness + groundedness) → a three-model comparison (Claude / GPT / Gemini). Reproduce with the commands in results.html.
B
Observability: traces land in AX; a deliberate retriever failure (drop corpus docs) makes the groundedness judge score those traces low; root-cause in the waterfall (empty RETRIEVER span → fabricated answer); fix → score recovers. The before/after delta is the story.

Using this repo · run it

Run the agent in ~2 minutes

# 3.12 for wheel compatibility
uv venv --python 3.12 && source .venv/bin/activate
uv pip install -r requirements.txt
cp .env.example .env        # fill OPENROUTER_API_KEY + ARIZE_SPACE_ID/ARIZE_API_KEY

python -m src.run                                  # default demo question
python -m src.run "How do I run an experiment with an LLM judge?"
python -m src.run --model gpt "..."            # swap provider (model-agnostic demo)

Model auth is a single OpenRouter key (reaches Claude/GPT/Gemini through one OpenAI-compatible endpoint). The tracing backend is chosen by TRACING_BACKEND=ax|phoenix — only src/tracing.py knows the difference.

▲ secrets

The repo is git-clean — .env and .venv are gitignored; the evidence screenshots ship. Copy .env.example to .env and add your own keys to run.
